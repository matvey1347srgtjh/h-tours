### Сценарий #1: Создание поста

Цель: Проверить функциональность добавления новой записи в базу данных через форму.

- Предпосылки: Пользователь открыл страницу создания поста.
- Шаги:
  1. Перейти на /post/new/.
  2. Заполнить поля title, text и author_name.
  3. Нажать кнопку "Опубликовать".
- Ожидаемый результат: Система перенаправляет на главную страницу, в базе данных количество постов увеличивается на 1.

### Сценарий #2: Редактирование записи

Цель: Убедиться, что изменения в существующем посте сохраняются корректно.

- Предпосылки: В базе данных существует пост с id=1.
- Шаги:
  1. Перейти на /post/1/edit/.
  2. Изменить текст в поле title.
  3. Сохранить форму.
- Ожидаемый результат: Заголовок поста в базе данных обновлен на новый.

### Сценарий #3: Система комментариев

Цель: Проверить связь "Один ко многим" между постом и комментариями.

- Предпосылки: На главной странице отображается существующий пост.
- Шаги:
  1. В блоке комментариев под постом ввести имя автора и текст.
  2. Нажать "Отправить".
- Ожидаемый результат: Комментарий отображается под постом, поле post_id комментария соответствует ID поста.

### Сценарий #4: Удаление контента

Цель: Проверить корректность удаления данных.

- Предпосылки: Выбран пост для удаления.
- Шаги:
  1. Перейти на страницу подтверждения удаления /post/1/delete/.
  2. Подтвердить действие (отправить POST-запрос).
- Ожидаемый результат: Пост удален, при попытке зайти на его страницу возвращается ошибка 404.
